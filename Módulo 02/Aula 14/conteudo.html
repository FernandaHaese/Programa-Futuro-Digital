Sass

Um sistema Sass (Syntactically Awesome Style Sheets) é um pré-processador CSS que adiciona funcionalidades avançadas ao CSS, como variáveis, aninhamento, mixins (funções) e herança, para tornar o código mais modular, reutilizável e fácil de manter. O código Sass é escrito numa linguagem que é convertida para CSS padrão para que possa ser lido e interpretado pelos navegadores web.

Um navegador não entende código Sass. Portanto, você precisará de um pré-processador Sass para converter o código Sass em CSS padrão. Esse processo é chamado de transpilação. Portanto, você precisa fornecer a um transpilador (algum tipo de programa) algum código Sass e, em seguida, obter de volta algum código CSS.

Variáveis

No Sass, podemos definir variáveis, que permitem definir valores um única vez e aplicá-los a diversos elkementos, facilitando ajustes posteriores. Podemos declarar uma variável utilizando o operador $, então passamos a propriedade, podendo ela ser uma cor, fonte, tamanho da fonte, largura ou altura, etc. variáveis:

Nesting

Nesting significado sintaxe aninhada, permitindo organizar os seletores de forma hierarquicá - assim como no código HTML, deixando o código muito mais limpo e legível

Mixin

A propriedade mixin permite que você crie um grupo de declarações CSS que pode ser reutilizado em todo o seu site. Portanto, primeiro criamos um propriedade @mixin name seguida pelos valores em {} e depois usamos a @include para permitir que você use (inclua) o mixin.

Funções

O Sass permite também que adicionemos funções em nosso código Css, uma excelente maneira de otimizar o código, impedir repetições e facilitar alterações. Para isso, o Sass utiliza o @funtion, @for, @if, @else, @each, @while ...

Herança

A @extend permite que você compartilhe um conjunto de propriedades CSS de um seletor para outro. A @extend é útil se você tiver elementos com estilos quase idênticos, mas que diferem apenas em alguns pequenos detalhes.
